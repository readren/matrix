# sbt Interactive Testing

Use this skill when you need to compile scala/java code efficiently within an active sbt session to avoid JVM startup overhead.

## Activation Criteria
- **Restriction**: This skill MUST NOT be used automatically or as a background task.
- **Permission**: Only activate this skill if the user provides an explicit imperative command.
- **Trigger Phrases**: After having touched the code base due to direct commands like "edit the code" or "generate code".
- **Confirmation**: If the user's intent is ambiguous, ask for permission before compiling.

## Environment
- SBT + Scala 3 (`scalaVersion := 3.8.3`, `sbt.version=1.11.5`).

## 1. Entering Interactive Mode
To start the sbt shell, execute `sbt` in the project root without any arguments.
- **Trigger**: The prompt will change to `> ` or `[project-name] $ `.
- Keep the sbt shell open to reuse it until you finish whatever you are doing.

## 2. Testing Command
Once inside the shell, use this command to compile:
- `compile`: Starts incremental compilation of the entire project main code.
- `Test / compile`: Starts incremental compilation of the entire project test tests.
- `<subprojectName> / compile`: Starts compilation of the specified subproject main code, and of all the subprojects it depends on.
- `<subprojectName> / Test / compile`: Starts compilation of the specified subproject test code, and of all the subprojects it depends on.
- `clean`: Deletes the files generated by previous compilations in all subprojects.

## 3. Shell Navigation Tips
- **Tab Completion**: Hit `<TAB>` to see available test class names.
- **History**: Use the Up/Down arrows to cycle through previous commands, or type `!` to see command history.
- **Exit**: Type `exit` or use `Ctrl+D` to leave the shell.

## Examples
- `consensus / Test / compile` (compiles test code for the consensus subproject).
